# SD Dragons FC — Club Dashboard

Performance dashboard for **all SD Dragons FC teams** competing in the
California Soccer League (Friday Youth divisions).

**Live URL:** https://cgrout.github.io/dragons-dashboard/

## What it covers

Nine Dragons teams across four age brackets:

| Bracket | Teams |
| --- | --- |
| **U7** (2019–2020) | SD Dragons Coed U7 |
| **U9** (2017–2018) | SD Dragons Coed U9 · SD Dragons Girls U9 |
| **U11** (2015–2016) | SD Dragons Coed U11 · SD Dragons Will U11 |
| **U13** (2013–2014) | SD Dragons Girls 2012 · SD Dragons Girls 2013 · SD Dragons 2014 · SD Dragons Will |

## What's in here

- `index.html` — the entire dashboard, single self-contained file. Embedded
  club crest (base64), all standings & match data baked in, inline CSS &
  JS. No external dependencies except Google Fonts.
- `README.md` — this file.

## Dashboard structure

- **Club Overview** — landing page with combined club record, all Friday
  matchups, "what's going right / what to watch" notes, and a sortable
  table of all nine teams.
- **Bracket tabs (U7 / U9 / U11 / U13)** — each shows a bracket overview
  (Friday matches, Dragons teams in that bracket as cards, top of the
  league table with Dragons highlighted) plus sub-tabs for each Dragons
  team in that bracket.
- **Per-team pages** — full analytics: rank, W-D-L, GD, points, PPG,
  form (last 5 real games), strength of schedule, record vs. better- and
  worse-ranked opponents, recent results, next-match prediction.

URL hash routing means every team page is deep-linkable, e.g.
`https://cgrout.github.io/dragons-dashboard/#u13/u13-will`.

## How to update the dashboard

The dashboard is a **snapshot** generated by Claude using data from
`californiasoccerleague.com`. To refresh it:

1. Ask Claude to regenerate the dashboard with current data.
2. Download the new `index.html` from the chat output.
3. In this repo on GitHub:
  - Click `index.html`
  - Click the pencil icon (Edit), paste the new contents, and commit, **OR**
  - From the repo root: "Add file" → "Upload files" → drag the new file
    (it will replace the existing `index.html`), then commit.
4. GitHub Pages re-deploys automatically within ~60 seconds.

The "SNAPSHOT · <date>" stamp in the dashboard header tells you which
data version is live.

## Methodology notes

- **Real games vs. forfeit games.** CSL counts forfeit wins (marked
  "NO GAME") as 5-0 victories in standings. The dashboard's per-game
  metrics (form, GF/g, GA/g, SoS) use only matches that were actually
  played. Standings totals (rank, W-D-L, points) include forfeits as
  the league does.
- **Strength of schedule** = average PPG of opponents faced in real
  games. League average is roughly 1.4. Lower = easier opposition.
- **Form (last 5 real)** excludes forfeits — it reflects how the team
  is actually playing.
- **Predictions** are commentary, not bets. Calibrated against opponent
  rank, current form, and SoS.
- The Friday Youth brackets mix boys, girls, and coed teams in the same
  standings. Rank position is league-wide, not gendered.

## Enabling GitHub Pages (one-time, if not already on)

1. Repo → **Settings** → **Pages**
2. **Source:** "Deploy from a branch"
3. **Branch:** `main`, folder `/ (root)`
4. Click **Save**
5. Wait ~60 seconds, then the URL appears at the top of Pages settings

## Optional improvements

- **Custom domain:** Settings → Pages → Custom domain. Requires a domain
  you own and a CNAME DNS record pointing at `cgrout.github.io`.
- **Auto-update via GitHub Actions:** A scheduled workflow could scrape
  the league site weekly, regenerate the data files, and rebuild
  `index.html`. Not currently set up — would convert this from a
  manually-refreshed snapshot into a true living dashboard.
